### **Визуализация процесса:**

```
ФАЙЛ ТАБЛИЦЫ ДО VACUUM:
┌─────────────────────────────────────┐
│ [LIVE] [DEAD] [LIVE] [DEAD] [LIVE]  │ ← 5 страниц данных
│ [DEAD] [DEAD] [LIVE] [DEAD] [LIVE]  │
└─────────────────────────────────────┘
Размер файла: 10 страниц

ФАЙЛ ТАБЛИЦЫ ПОСЛЕ ОБЫЧНОГО VACUUM:
┌─────────────────────────────────────┐
│ [LIVE] [FREE] [LIVE] [FREE] [LIVE]  │ ← 5 страниц данных
│ [FREE] [FREE] [LIVE] [FREE] [LIVE]  │ ← FREE = доступно для записи
└─────────────────────────────────────┘
Размер файла: 10 страниц (НЕ ИЗМЕНИЛСЯ!)

ФАЙЛ ТАБЛИЦЫ ПОСЛЕ VACUUM FULL:
┌─────────────────┐
│ [LIVE] [LIVE]   │ ← Данные сжаты
│ [LIVE] [LIVE]   │
│ [LIVE]          │
└─────────────────┘
Размер файла: 5 страниц (УМЕНЬШИЛСЯ!)
```

### **Что происходит при обычном VACUUM:**

```sql
-- PostgreSQL помечает страницы как доступные в Free Space Map
-- Новые INSERT'ы будут использовать эти освобожденные места

INSERT INTO users (name) VALUES ('New User');
-- ↑ Запишется в [FREE] место вместо расширения файла
```
