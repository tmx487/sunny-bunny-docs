1. На сайте Microsoft указано, что для класса модификатором доступа по умолчанию является [`internal`](https://learn.microsoft.com/en-us/dotnet/csharp/language-reference/keywords/class#remarks). Тогда почему в данном случае:

```c#
namespace TeacherResource.API.Common
{
    static class Routes
    {
        static class Auth
        {
            public const string Base = "auth";
            public const string Login = "login";
            public const string Register = "register";
        }
        static class Topic
        {
            public const string TopicList = "topics";
            public const string TopicByPersonId = "topics/{personId}";
            public const string TopicById = "topics/{topicId}";
        }
    }
}
```

верхний класс `Routes` виден в других файлах этой же сборки, а внутренние `Auth` и `Topic`, а также поля - нет? Внутренние типы становятся доступными только при **явном** указании модификатора `internal` при объявлении этих типов (при этом объявление `Routes` как `internal` не требуется):

```c#
namespace TeacherResource.API.Common
{
    static class Routes
    {
        internal class Auth
        {
            public const string Base = "auth";
            public const string Login = "login";
            public const string Register = "register";
        }
        internal static class Topic
        {
            public const string TopicList = "topics";
            public const string TopicByPersonId = "topics/{personId}";
            public const string TopicById = "topics/{topicId}";
        }
    }
}
```
